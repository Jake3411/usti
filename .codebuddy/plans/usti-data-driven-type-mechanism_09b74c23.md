## 用户需求

- 基于真实学生数据，将大学生学习行为分成若干典型类型，并通过 10 道问卷题将用户归入这些类型之一。
- 问卷答案与类型之间的对应关系必须主要由数据驱动，而不是纯主观拍脑袋规则；分类结果要能随答案变化而明显变化，而不是几乎总落在同一类型。
- 类型数量不必固定为 6，可以按数据情况调整；现有 C0–C5 名称和文案来源于之前的数据分析，希望新机制仍能支持类似风格的类型描述。
- 希望能看到大致的特征权重或重要性，用于解释「哪些特征更影响类型划分」。

## 产品概述

- 产品提供一个基于真实学生行为数据构建的「大学生类型指示器」，通过在线问卷快速评估用户在学习投入、学习方式、校园参与等维度上的特征。
- 系统先在数据集中挖掘若干具有代表性的学生类型，再将问卷答案映射到数据特征空间中，与这些典型类型进行匹配，输出最接近的类型及解释。
- 结果页面展示用户当前类型、该类型的文字说明、潜在问题、改进建议，以及与同类样本的对比信息和关键特征提示。

## 核心功能

- 从给定数据集中自动挖掘若干「典型大学生类型」，形成每一类的统计特征与文字概括。
- 将 10 道问卷每个选项转换为与数据分布对齐的特征值，使问卷用户在特征空间中的位置真实反映其行为倾向。
- 基于数据构建简单且可解释的类型预测机制，对问卷用户给出类型预测，并提供各类型匹配程度和特征重要性提示。
- 结果页面展示：最终类型（如 C0–C5）、对应文案、同类典型样本示例、聚类概览表，以及用户自身特征与不同类型特征的对比线索。

## Tech Stack 选择

- 语言与环境：Python。
- Web 展示：现有 Streamlit 应用保持不变。
- 数据与建模：延续当前使用的数值/类别特征工程与聚类框架，在此基础上增加一个简单的多类分类模型用于类型预测和权重解释。

## 实现思路概述

- 继续使用无监督聚类在数据集中找到若干行为模式簇，用于定义「典型大学生类型」及其统计特征。
- 将问卷选项到特征值的映射改为**基于数据分布的数值校准**，使 A/B/C 对应到数据中的高/中/低区间；确保问卷用户落在真实数据的合理范围。
- 在得到每个样本的类型簇标签后，训练一个简单的多类分类模型，将「特征 → 类型」的关系显式建模，并利用模型系数或重要性指标提供可解释的特征权重。
- 预测时，不再直接依赖聚类最近中心，而是用该分类模型对问卷用户的特征行进行类型预测，并返回类型概率分布等解释信息。

### 关键技术决策与权衡

- 聚类算法：保持当前聚类方式（例如 KMeans）用于在数据上挖掘类型，避免破坏已有的类型结构和文案基础；可根据轮廓系数等指标在一个合理的 K 范围内自动选择类型数量。
- 问卷映射：弃用固定手写数值，改用数据驱动的分位数/区间映射，使问卷生成的特征更贴近真实样本分布，避免「几乎全部聚到同一类」。
- 分类模型：选择线性多类分类器（如一对多逻辑回归），在预处理后的特征空间上学习「特征→类型」边界：
- 优点：决策边界简单，系数可直接解读为特征重要性；在维度较低、样本量适中的情况下表现稳定。
- 若后续发现线性模型表达力不足，可增加正则、特征交互或改用树模型，但第一版优先保证可解释性。
- 类型数量：短期内可以维持 6 类以兼容现有 C0–C5 文案，同时在训练接口中保留调节 K 范围的能力，为后续扩展更多类型预留空间。

### 性能与可靠性

- 训练复杂度：
- 聚类：随样本数和特征维度线性增长，对当前规模数据开销可控。
- 分类：线性多类模型在当前特征维度下训练成本低，可在应用初始化或离线阶段完成。
- 预测性能：
- 每次预测只需一次预处理 + 一次线性模型推断，时间复杂度为特征维度的线性量级，对单用户交互即时性影响极小。
- 可靠性与可维护性：
- 所有模型、预处理器、类型描述都集中封装在 USTIArtifacts 中，前端只依赖统一的预测接口，便于后续替换内部实现。
- 通过增加对问卷映射和预测结果的调试展示（如不同答题组合的类型分布），帮助快速发现和修正异常行为。

## 实现细节说明

### 构建数据驱动的问卷映射

- 在训练阶段，从原始数据 `student_performance_grade.csv` 中计算每个数值特征的分布统计（如分位数、均值、标准差）。
- 为每一个与问卷对应的特征定义：
- A 选项：对应该特征在数据中的高位区间（例如 75%–90% 分位）。
- B 选项：对应中位附近区间。
- C 选项：对应低位区间。
- 对于某些含义颠倒的特征（例如「刷屏时间」中 C 实际是较低水平），映射顺序相应反转，使语义与数据分布一致。
- 将这些映射规则封装在新的映射函数中，替换现有的固定数值表，从而保证问卷生成的「伪样本」与真实学生特征空间对齐。

### 引入可解释的类型预测模型

- 在完成聚类后：
- 利用训练数据的特征以及对应的簇标签，构建多类分类训练数据集。
- 在预处理后的特征空间上训练线性多类分类模型，用聚类簇标签作为监督信号。
- 在 USTIArtifacts 中增加：
- 分类模型实例；
- 每个类型的特征权重或重要性摘要（例如按特征分组求系数范数，选出对该类型最具区分力的若干特征）。
- 预测时：
- 对问卷答案生成特征行，应用同一预处理器；
- 使用分类模型输出对各类型的预测概率，取概率最高者作为最终类型，同时保留完整的概率分布用于展示或调试；
- 仍可保留聚类最近中心结果作为参照，辅助分析「问卷生成的点与真实簇中心的距离」。

### 前端展示与交互调整

- 在现有类型结果展示区域中增加：
- 各类型预测概率表或条形图，让用户看到当前答题与其他类型的接近程度；
- 本次预测中最关键的几项特征提示，例如「与典型 C1 相比，你的睡眠时长明显偏低」。
- 保持现有 C0–C5 文案展示不变，但内部不再强依赖「cluster id = 文案 id」的硬编码关系，而是通过训练结果和映射配置显式指定。

## 架构设计

- 整体保持分层结构：
- 数据与模型层（`src/usti_pipeline.py`）：负责数据加载、预处理、聚类、分类训练、类型解释封装。
- 表现层（`app.py`）：负责问卷渲染、调用预测接口、展示类型结果与解释信息。
- 核心数据流：
- 训练阶段：CSV 数据 → 清洗选特征 → 构建预处理器 → 聚类得到簇标签与簇特征 → 基于簇标签训练分类模型 → 生成 USTIArtifacts。
- 预测阶段：问卷答案 → 映射到特征行 → 预处理 → 分类模型预测类型及概率 → 查表获取类型文案与解释 → 前端展示。
- 后续扩展：
- 可在 USTIArtifacts 中追加更多解释性输出（如按类型的特征均值对比表），而无需修改前端调用接口。
- 若将来增加类型数量或引入新的特征，只需调整训练管线和问卷映射逻辑。
